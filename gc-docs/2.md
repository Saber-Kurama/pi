这三个类型都在 **`packages/ai/src/types.ts`**（前两个）和 **`packages/agent/src/types.ts`**（`AgentLoopConfig`）里，关系是：

**`StreamOptions` → `SimpleStreamOptions`（+ reasoning）→ `AgentLoopConfig`（extends SimpleStreamOptions + agent loop 专用字段）**

---

## `StreamOptions`（`packages/ai/src/types.ts`）

所有流式/补全请求的**通用 HTTP + 生成参数**，各 provider 不认识的字段会忽略。

| 字段 | 类型 | 说明 |
|------|------|------|
| `temperature?` | `number` | 采样温度 |
| `maxTokens?` | `number` | 生成长度上限（具体字段名由 provider 映射） |
| `signal?` | `AbortSignal` | 取消请求 |
| `apiKey?` | `string` | 本次请求的 API key（可覆盖默认） |
| `transport?` | `Transport` | `"sse" \| "websocket" \| "websocket-cached" \| "auto"`，多传输方式时优先选 |
| `cacheRetention?` | `CacheRetention` | `"none" \| "short" \| "long"`，提示缓存保留策略，默认 `"short"` |
| `sessionId?` | `string` | 会话级缓存/路由等（provider 支持才用） |
| `onPayload?` | `(payload, model) => ...` | 发请求前检查或改写 body |
| `onResponse?` | `(response, model) => void` | 收到 HTTP 响应后、读 body 前回调 |
| `headers?` | `Record<string, string>` | 额外 HTTP 头（与默认合并） |
| `timeoutMs?` | `number` | 请求超时（毫秒） |
| `maxRetries?` | `number` | 客户端重试次数 |
| `maxRetryDelayMs?` | `number` | 服务端要求长等待时的上限（默认 60000，`0` 表示不封顶） |
| `metadata?` | `Record<string, unknown>` | 透传元数据（如 `user_id`） |

---

## `SimpleStreamOptions`（`extends StreamOptions`）

在 **`StreamOptions`** 上增加「推理 / thinking」相关选项，供 **`streamSimple` / `completeSimple`** 及 agent loop 底层调用。

| 字段 | 类型 | 说明 |
|------|------|------|
| （继承 `StreamOptions` 全部字段） | | |
| `reasoning?` | `ThinkingLevel`（pi-ai） | 这里是 **`"minimal" \| "low" \| "medium" \| "high" \| "xhigh"`**，没有 `"off"`；关 thinking 在 agent 层用 `thinkingLevel === "off"` 时不传 `reasoning` |
| `thinkingBudgets?` | `ThinkingBudgets` | 各档位的 token 预算（仅部分按 token 计费的 provider 使用） |

`ThinkingBudgets`（同文件）：`minimal?`、`low?`、`medium?`、`high?` 等数字字段。

---

## `AgentLoopConfig`（`packages/agent/src/types.ts`，`extends SimpleStreamOptions`）

= **`SimpleStreamOptions` 全部字段** + **必选 `model`** + **必选 `convertToLlm`** + **agent 循环语义**（工具、队列、上下文变换等）。

### 来自 `SimpleStreamOptions`（含 `StreamOptions`）

即：`temperature`、`maxTokens`、`signal`、`apiKey`、`transport`、`cacheRetention`、`sessionId`、`onPayload`、`onResponse`、`headers`、`timeoutMs`、`maxRetries`、`maxRetryDelayMs`、`metadata`、`reasoning`、`thinkingBudgets` —— 都会传给底层 `streamSimple` 一类调用。

### Agent 循环专有字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `model` | `Model<any>` | **必选**，当前轮使用的模型（api/provider/id 等） |
| `convertToLlm` | `(AgentMessage[]) => Message[] \| Promise<...>` | **必选**，把 `AgentMessage` 转成 provider 能吃的 `Message[]`；**不能 throw** |
| `transformContext?` | `(messages, signal?) => Promise<AgentMessage[]>` | 在 `convertToLlm` 之前对整条上下文做裁剪/注入；**不能 throw** |
| `getApiKey?` | `(provider) => string \| undefined \| Promise<...>` | 每轮动态取 key（OAuth 等）；**不能 throw**，无 key 时 `undefined` |
| `shouldStopAfterTurn?` | `(ShouldStopAfterTurnContext) => boolean \| Promise<...>` | 某 turn 完全结束且已 `turn_end` 后调用；`true` 则发 `agent_end` 并退出，**不再**拉 steer/follow-up |
| `getSteeringMessages?` | `() => Promise<AgentMessage[]>` | 当前 assistant + tools 跑完后注入「插队」用户消息；**不能 throw**，无则 `[]` |
| `getFollowUpMessages?` | `() => Promise<AgentMessage[]>` | 无 tool、无 steer 时拉「后续」消息再开一轮；**不能 throw**，无则 `[]` |
| `toolExecution?` | `ToolExecutionMode` | `"sequential"` 或 `"parallel"`（默认 parallel） |
| `beforeToolCall?` | `(ctx, signal?) => BeforeToolCallResult \| Promise<...>` | 校验后可 `{ block: true, reason? }` 阻止执行 |
| `afterToolCall?` | `(ctx, signal?) => AfterToolCallResult \| Promise<...>` | 执行后可改 content/details/isError/terminate |

---

## 关系一句话

- **`StreamOptions`**：纯「调模型 HTTP/生成」层。  
- **`SimpleStreamOptions`**：在上面加 **reasoning / thinkingBudgets**。  
- **`AgentLoopConfig`**：再在上面加 **model + transcript↔LLM 映射 + 工具与 steer/follow-up 循环策略**。
